### Installation & Quick Start

1. **Clone the repository:**
   ```bash
   git clone https://github.com/Maximbrg/model_evaluation.git
   cd model_evaluation
   ```

2. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Configure API Key:**
   Open `multi_agent/config.yaml` and replace `"your-openai-api-key-here"` with your actual OpenAI API Key.

4. **Run the Framework:**
   ```bash
   python multi_agent/new_framework.py
   ```
   *(Results will be saved in a dynamically timestamped folder inside `outputs/`)*

### Architecture overview for `new_framework.py`

`new_framework.py` acts as the primary **LangGraph Orchestrator** for Phase 1. It wires together specific nodes (functions) and edges (transitions) to create a cyclical state machine. It does not contain the LLM calls itself; instead, it delegates to three highly modular files:

**1. `framework_state.py`**
* **Responsibility:** Defines the `AgentState` schema using Python's `TypedDict`. 
* **Why it matters:** This acts as the shared memory payload that is passed continuously between nodes. It tracks the `api_key`, `case_model`, `iteration` count, generated `guidelines`, and the ongoing `syntax_history`.

**2. `framework_prompts.py`**
* **Responsibility:** Centralizes all system and user prompt generation.
* **Why it matters:** It separates the "text generation logic" from the "routing logic". It formats the templates, injects the domain descriptions, and structures the JSON requests so the LLMs output strict, parsable decisions.

**3. `framework_nodes.py`**
* **Responsibility:** Contains the actual LangGraph node definitions and the code that executes OpenAI API calls.
* **Key Nodes:**
  * **`ask_agent_1_for_template`**: Agent 2 determines if it needs a structural CSV template based on the domain description.
  * **`create_evaluation_template`**: Agent 1 (Interpreter) queries its strict metamodel to output the required CSV syntax.
  * **`draft_guidelines_constraints`**: Agent 2 (Analyzer) drafts the evaluation criteria base on the problem and the template.
  * **`refine_guidelines_constraints`**: Agent 2 (serving as the Checker) evaluates the draft against the case model. Crucially, it decides whether to transition backward to Agent 1 (`answer_syntax_questions`), loop backward to refine the `draft`, or transition forward to finalize (`receive_guidelines_constraints`).

**How they work together:**
If a user runs the main framework, `new_framework.py` initializes the `AgentState` with inputs from `config.yaml` and drops it into `trigger_agent_2`. The graph then autonomously routes the state through `framework_nodes.py`, leveraging the dialogue generated by `framework_prompts.py`, looping until the guidelines are deemed satisfactory or the iteration limit is reached. Detailed logs of exactly what happens between nodes are saved automatically to `stage_interactions.csv`.
